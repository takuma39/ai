# リサーチパターン

research-agent・fact-checker が調査時に従う、情報収集の型と優先順位。

---

## 情報源の優先順位

| 優先度 | 情報源 | 信頼度 | 例 |
|---|---|---|---|
| 1 | 公式ドキュメント | 高 | docs.anthropic.com, docs.github.com |
| 2 | 公式ブログ・リリースノート | 高 | anthropic.com/blog, github.blog |
| 3 | GitHubリポジトリ（README, CHANGELOG） | 高 | github.com/anthropics/claude-code |
| 4 | 信頼性の高いテックメディア | 中 | The Verge, TechCrunch, InfoQ |
| 5 | カンファレンス発表・公式動画 | 中 | YouTube公式チャンネル |
| 6 | 個人ブログ・Qiita・Zenn | 低 | 裏付けが必要 |
| 7 | SNS（X/Twitter） | 低 | 速報性はあるが要検証 |

---

## 検索クエリパターン

### ツールの最新バージョン確認

```
英語: "[tool name]" latest version release 2025 OR 2026
日本語: "[ツール名]" 最新バージョン リリース
```

### 機能追加・変更の確認

```
英語: "[tool name]" changelog OR "release notes" OR "what's new"
日本語: "[ツール名]" 新機能 アップデート
```

### 廃止・非推奨の確認

```
英語: "[tool name]" deprecated OR discontinued OR "end of life"
日本語: "[ツール名]" 廃止 非推奨 サポート終了
```

### ベストプラクティスの調査

```
英語: "[tool name]" best practices 2025 OR 2026
日本語: "[ツール名]" ベストプラクティス 活用事例
```

### 比較・選定の調査

```
英語: "[tool A]" vs "[tool B]" comparison 2025 OR 2026
日本語: "[ツールA]" "[ツールB]" 比較 違い
```

---

## 検索時のルール

1. **英語・日本語の両方で検索する** — 英語の方が最新情報が早い傾向
2. **日付フィルタを活用する** — 6ヶ月以内の情報を優先
3. **複数の情報源でクロスチェック** — 1つの情報源だけで判断しない
4. **公式 > 非公式** — 矛盾する場合は常に公式情報を採用
5. **「情報が見つからない」も有用な結果** — Unverifiable として報告する

---

## ファクトチェック対象の分類

| カテゴリ | チェック方法 | 例 |
|---|---|---|
| バージョン番号 | 公式リリースノート | "Claude 3.5 Sonnet" → 最新は? |
| 日付 | 公式ブログ・プレスリリース | "MCP移管: 2025年3月" → 正しいか? |
| 数値・仕様 | 公式ドキュメント・料金ページ | "コンテキスト200K" → 現在の値は? |
| 組織名 | 公式サイト | "Linux Foundation" → 正式名称か? |
| 機能説明 | 公式ドキュメント | "Cursor Agent Mode" → 実在するか? |
| URL | WebFetchで確認 | リンク切れチェック |

---

## 調査結果の信頼度判定基準

### 高（Verified）
- 公式ドキュメントで確認済み
- 2つ以上の信頼できる情報源で一致

### 中（Likely Correct）
- 1つの信頼できる情報源で確認
- 公式情報は未確認だが矛盾する情報もない

### 低（Needs Verification）
- 非公式情報源のみ
- 情報が古い可能性がある（6ヶ月以上前）
- 情報源間で矛盾がある
