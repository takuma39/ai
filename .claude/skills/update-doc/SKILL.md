AI駆動開発.md の既存セクションを最新情報で更新します。

対象セクション: $ARGUMENTS

`/add-section` が新規セクションを追加するのに対し、このコマンドは**既存セクションの情報鮮度を維持**するために使用します。

## 実行手順

### ステップ1: 対象セクションの特定

`AI駆動開発.md` を読み、`$ARGUMENTS` に対応するセクションを特定する。
- 見つからない場合はユーザーに正確なセクション名を確認する
- 複数のセクションが該当する場合は全て対象とする

### ステップ2: 並列調査

**research-agent** と **fact-checker** をバックグラウンドで並列実行する（`run_in_background: true`）:

- **research-agent**: 対象セクションのトピックについて最新情報を収集
  - Context7 MCP が利用可能な場合は公式ドキュメントを優先的に参照
  - 追加すべき新情報・削除すべき古い情報を特定
- **fact-checker**: 現在の記述の正確性を検証
  - バージョン番号・日付・数値・ツール名を重点的に確認
  - Outdated / Incorrect な記述をリストアップ

### ステップ3: セクション更新

**doc-writer** サブエージェントを呼び出し、以下の方針で更新する:
- 文体・構成・見出し階層は維持する
- 事実の誤り・古い情報のみを修正する
- 新しい情報を適切な位置に追加する
- 廃止された情報を削除する
- 変更が最小限になるよう差分適用（大幅な書き直しはしない）

### ステップ4: レビュー

**doc-reviewer** サブエージェントを呼び出し、更新後のセクションの品質チェックを実行する。

### ステップ5: 完了報告

以下の形式で報告する:

```
## 更新完了: [セクション名]

### 変更内容
- 修正した誤り: [リスト]
- 更新した情報: [リスト]（例: バージョン番号を X から Y に更新）
- 追加した情報: [リスト]
- 削除した情報: [リスト]

### レビュー結果
（doc-reviewer の主要指摘）
```

## 注意事項

- `$ARGUMENTS` が空の場合は「使用ツール」セクション全体を対象とする（月次メンテ用途）
- 文体・構成の大幅な変更が必要な場合はユーザーに確認を求める
- 更新後は `git diff AI駆動開発.md` で変更内容を確認することを案内する
