スキルファイル（agent / command / knowledge）を新規作成または更新します。

引数: $ARGUMENTS
形式: `[タイプ] [名前と目的の説明]`
例:
- `agent 最新情報を自動収集するエージェント update-watcher`
- `command 使用ツールセクションを一括更新するコマンド update-tools`
- `knowledge MCP設定のベストプラクティス集 mcp-patterns`

## 実行手順

### ステップ1: 要件整理

`$ARGUMENTS` から以下を特定する：
- **タイプ**: `agent` / `command` / `knowledge` のいずれか（明示されていない場合は目的から推定）
- **名前**: ファイル名に使用（ケバブケース）
- **目的**: このスキルが何をするか

### ステップ2: 既存スキルの確認

**skill-architect** サブエージェントを呼び出し、以下を実行させる：

1. 既存のスキルファイルを全て読んでパターンを把握
2. 要件に合った雛形を生成
3. 適切なディレクトリ（`agents/[name].md` / `skills/[name]/SKILL.md`）にファイルを作成
4. `CLAUDE.md` の対応テーブルを更新

### ステップ3: README の同期

**readme-updater** サブエージェントを呼び出し、`README.md` を最新状態に同期する。

### ステップ4: 完了報告

以下を報告する：
- 作成/更新したファイルのパス
- ファイルの主要な内容（要約）
- CLAUDE.md の更新箇所
- README の更新箇所

## 注意事項

- 同名のファイルが既に存在する場合は「更新モード」として差分を適用する
- タイプが不明な場合は目的から最適なタイプを選択する（確認は不要）
- 作成後は必ず `/review` でレビューすることを推奨メッセージで案内する
