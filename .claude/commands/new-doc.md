ゼロから新しいドキュメント（またはAI駆動開発.md の大規模セクション群）を生成します。

テーマ: $ARGUMENTS

## 実行手順

### ステップ1: 要件整理

ユーザーに以下を確認してください：
- ドキュメントの目的と対象読者
- カバーすべきトピック・範囲
- 既存の `AI駆動開発.md` への追加か、独立ドキュメントか
- 想定ボリューム（セクション数の目安）

### ステップ2: リサーチ

**research-agent** サブエージェントを呼び出し、「$ARGUMENTS」に関連する以下を調査：
- 主要ツール・フレームワークの最新情報
- ベストプラクティスと業界動向
- 比較すべき選択肢

### ステップ3: 構成設計

調査結果をもとに、ドキュメントの構成案を作成してください：

```
## [大セクション1]
### [サブセクション1.1]
### [サブセクション1.2]
## [大セクション2]
...
```

構成案をユーザーに提示し、承認を得てから次に進む。

### ステップ4: 執筆

**doc-writer** サブエージェントを呼び出し、承認された構成に基づいてセクションを順次執筆。

執筆時の参照ファイル：
- `.claude/skills/doc-standards.md` — 書式・文体ルール
- `.claude/skills/ai-usecase-templates.md` — テンプレート選択
- `.claude/skills/research-patterns.md` — 情報の信頼度基準

### ステップ5: レビュー

以下のエージェントで段階的にレビュー：

1. **format-checker** — 書式チェック（並列実行可）
2. **doc-reviewer** — 総合品質レビュー（並列実行可）
3. **fact-checker** — 事実関係の検証（並列実行可）
4. **senior-engineer-reviewer** — ベテラン視点レビュー

### ステップ6: 修正・完成

レビュー結果の指摘事項を **doc-writer** で修正し、完成版を出力。

## 注意事項

- 各ステップの結果をユーザーに報告し、次に進む前に確認を取ること
- `CLAUDE.md` のドキュメント規約に厳密に従うこと
- 大量のセクションを一度に書く場合は、3〜5セクションずつ区切って執筆→レビュー→修正を繰り返すこと
- すべて日本語で出力すること
